Android项目中值得注意的编码规范(一)

发表于 2016-04-13   |  

目前开发Android的主流语言为Java，本文为作者针对Java开发语言以及在开发Android项目过程中碰到的一些编码规范问题做出以下整理

1.命名规范

Android开发的过程中好的命名规范显得尤为重要，好的包命名规范可以使项目代码样式统一，使程序有良好的可读性

1.类命名

类名（首单词字母大写）

1.activity类

命名以Activity为后缀

2.fragment类

命名以Fragment为后缀

3.service类

命名以Service为后缀

4.adapter类

命名以Adapter为后缀

5.接口实现类

命名以Impl为后缀

6.工具类

命名以Util为后缀

2.常量命名

常量名（全部大写 ，常加下划线）

3.方法命名

方法名（首字母小写，字母开头大写）

4. 变量名

全局变量（首字母以小写m开头）

局部变量（小写）

2.控件缩写

在代码中使用到的控件应当遵守以下规范

控件 缩写
TextView Txt
EditText Edt
Button Btn
ImageButton Ibtn
ImageView Img
ListView List
RadioGroup Group
RadioButton Rbtn
ProgressBar Progress
SeekBar Seek
CheckBox Chk
Spinner Spinner
TableLayout Table
TableRow Row
LinearLayout Llayout
RelativeLayout Rlayout
ScrollView Scroll
SearchView Search
TabHost Host
TabWidget Widget

3.布局缩写

布局的xml文件应该遵循如下规范

布局 命名
Activity 以activity_
Fragment 以fragment_
Dialog 以dialog_
ListView的item 以itemlist
GridView的item 以item_grid
ListView的HeaderView 以header_list
ListView的FooterView 以footer_list
